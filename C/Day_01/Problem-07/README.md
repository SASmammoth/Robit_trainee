## 과제7 간단한 고찰.

_본 프로그램은 n과 r을 입력 받아 순열, 중복 순열, 조합, 중복 조합을 출력하는 프로그램이다._

팩토리얼 함수를 사용할 수 없기에, n!, (n-r)! (n+r-1)! 같은 값을 미리 변수로 만들어 저장하였다. 또한 제곱에 관한 함수도 사용할 수 없기에, 반복문을 이용하여 구현하였다.

int형 자료형은 약 13! 이상의 값을 저장할 수 없기에, 가장 빠르게 상승하는 (n+r-1)! 을 계산할 때 13! 이상의 값을 계산하여야 하면 예외 처리를 통해 프로그램을 종료하였다. 물론 수학적인 방법이나 unsigned int 나 long long처럼 더 큰 자료형을 사용하면 더 큰 값도 계산할 수 있지만, 팩토리얼은 기하급수적으로 증가하여 딱히 큰 도움이 안될 거라 생각하였다.

팩토리얼 한계 값을 하드 코딩한 이유는, 피보나치와는 다르게 점화식이 곱셈 연산이라 오버플로우가 2번 이상 일어난다면 두 수의 크기 비교 만으로는 정상 동작 한 거 처럼 보일 수도 있기 때문이다.
